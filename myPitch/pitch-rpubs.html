Basic ideas

To display a surface plot in 3D, we need a matrix with the values along one axis for each vertex of a grid along the 2 other axes.

  • that's how the volcano data come: a matrix of altitudes.
  • To display the water with the volcano, we just gave to the plot a second matrix, with same dimensions and a constant Z value (water level), modulated or not with waves.

Simple calculations

(as X & Y are in same units as Z)

  • Volume of water: Z difference when water is above terrain
  • volume of emerged land: Z difference when terrain is above water

Waves simulation (1)

This very basic simulation assumes a plane wave front with no diffraction at the volcano: \[ z=a\cdot\cos(\vec{k}.\vec{r}+\omega{t}+\varphi) \] Where \( z \) is the height of the wave at point \( \vec{r} \) and time \( t \).

The parameters of the wave front are:

  • \( a \): amplitude,
  • \( \vec{k} \): wave vector (depending on direction \( \theta \) and wave length \( \lambda \)),
  • \( \omega \): time pulsation (not used in the simulation),
  • \( \varphi \): phase.

Waves simulation (2)

Here is a simple way to build a 2D image of a wave using this equation: 

# local grid and time
x=1:50; y=1:100; t=0
# Wave space parameters
a=5; theta = 70/360*2*pi; lambda=20
k=2*pi/lambda
# Wave time and phase parameters
# ...not very useful at t=0...
freq=0.1; omega=2*pi/freq
phi=0/360*2*pi
# Wave function
wave <- function(x, y) {a * cos(k * 
        (cos(theta)*x+sin(theta)*y)+ 
                omega*t + phi)
}

image(x, y, z = z <- outer(x, y, "wave"))
